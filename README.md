这是一个由STM32下位机和QT上位机构成的雷达扫描及定位系统，想复刻这套系统需要有STM32单片机开发经验，QT使用经验，嘉立创PCB绘制能力，也要能熟练使用git命令。

## 快速开始

如果你已经有一套焊接好的硬件并烧录了下位机，只想运行上位机看效果：

```bash
# 连接串口后，直接运行（默认 COM8）
QRadar.exe

# 或者指定你的串口号
QRadar.exe COM3
QRadar.exe COM7
QRadar.exe COM10
```

命令行参数只接受一个，就是串口号。不传参数默认连 COM8。串口拔掉后会一直尝试重连，插回去就行，不用重启程序。

## 系统概述

本系统模拟真实雷达的扫描与目标定位功能。下位机以 STM32F103C8T6 为主控，驱动舵机带动超声波传感器在 0°~180° 范围内往复扫描，实时采集距离数据并通过串口发送给上位机。上位机使用 Qt 绘制雷达显示界面，以极坐标方式呈现扫描线、检测点和最近威胁目标。

### 工作流程

```
[舵机带动HC-SR04扫描] → [STM32采集距离] → [串口发送 角度,距离,量程]
        ↕
[上位机QRadar] ← 解析数据 → 极坐标绘制 → 威胁检测 → 屏幕显示
```

## 功能特性

- **实时扫描显示**：60fps 动画刷新，扫描线平滑跟随舵机角度
- **检测点余辉**：2.5 秒渐隐轨迹，模拟雷达 PPI 显示效果
- **威胁目标跟踪**：每个 180° 扫描周期内自动标记最近距离目标，红色十字高亮
- **多档量程**：支持 1~10 档量程切换，单档 64cm，最大量程 640cm
- **自动重连**：串口断开后自动重连，无需手动干预
- **无效数据过滤**：硬件侧超出量程的读数（500/505/510）自动滤除

## 硬件组成

| 组件 | 型号/参数 |
|------|----------|
| 主控 | STM32F103C8T6 |

## 下位机（STM32_Radar）

STM32_Radar 是下位机文件夹，可以使用 Keil5 进行编译烧录，主控使用 STM32F103C8T6。

- 舵机 PWM 控制超声波传感器在 0°~180° 之间往复扫描
- HC-SR04 发射超声波脉冲，根据回波时间计算距离
- 串口输出格式：`角度,距离,量程\n`（如 `90,45,3`）
- 无效读数（超出量程或无回波）发送 500/505/510 供上位机过滤

## 上位机（QRadar）

QRadar 是上位机文件夹，我用的版本是 Qt 5.4.0，编译器为 MinGW 4.9.1 32-bit。

### 安装依赖

| 依赖 | 版本 | 获取方式 |
|------|------|---------|
| Qt | 5.4.0 | [Qt Archive](https://download.qt.io/archive/qt/5.4/5.4.0/) |
| MinGW | 4.9.1 32-bit（Qt 自带） | - |
| STM32 HAL 库 | Keil5 自带 | - |

### 编译

```bash
# 1. 设置 Qt 环境变量（根据实际安装路径调整）
export PATH="/e/Software/qt/Tools/mingw491_32/bin:/e/Software/qt/5.4/mingw491_32/bin:$PATH"

# 2. 编译 Release 版本
cd QRadar
qmake QRadar.pro -spec win32-g++ "CONFIG+=release"
mingw32-make -f Makefile.Release

# 3. 编译 Debug 版本（开发调试用）
qmake QRadar.pro -spec win32-g++ "CONFIG+=debug"
mingw32-make -f Makefile.Debug
```

### 运行

```bash
# 默认连接 COM8
./release/QRadar.exe

# 指定其他端口
./release/QRadar.exe COM3
./release/QRadar.exe COM5
./release/QRadar.exe COM9
```

串口号不区分大小写（com3 和 COM3 都可以），只支持标准 COM 端口编号（COM1 ~ COM255）。

### 通信协议

串口波特率 115200，每行数据格式为 `angle,distance,range`：

| 字段 | 范围 | 说明 |
|------|------|------|
| angle | 0~180 | 舵机当前角度，0 对应物理右侧，180 对应物理左侧 |
| distance | 浮点数 | 超声波测距值，单位 cm |
| range | 1~10 | 当前量程档位，显示范围 = range × 64 cm |

### 数据示例

```
0,85.3,3
45,120.7,3
90,510,3       ← 510 表示无回波，上位机会过滤
135,62.1,3
180,500,3       ← 500 表示超出量程，上位机会过滤
```

## 注意事项

- HC-SR04 的有效探测距离约 2cm~450cm，精度约 3mm，但实际精度受温度和反射面材质影响
- 舵机 SG90 的实际转动范围可能略小于 180°，可用编码器校准（下位机已集成校准逻辑）
- 上位机靠近舵机的串口重新插拔一下即可实现串口重连
- PCB 文件在 `PCB/` 目录下，使用嘉立创 EDA 直接打开

## 复刻须知

1. 准备好 STM32F103C8T6 最小系统板、HC-SR04、SG90 舵机、CH340 串口模块
2. 使用 Keil5 编译烧录下位机程序，确认串口能收到 `角度,距离,量程` 格式数据
3. 安装 Qt 5.4.0 + MinGW 4.9.1 32-bit 工具链
4. 编译上位机，连接硬件后运行
